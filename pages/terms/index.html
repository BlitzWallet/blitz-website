12. WebView Processing System

12.1 System Overview

The App utilizes a proprietary secure Processing System that operates through a sandboxed WebView Component to handle computationally intensive wallet operations without impacting application performance.

12.2 Seed Transmission

Critical Technical Disclosure:

To initialize your wallet, the App must perform a one-time, in-memory transmission of your Seed from your device's secure storage into the WebView Component during each wallet session. This transmission is necessary to enable wallet functionality including:

  • Sending and receiving payments
  • Getting wallet balance
  • Getting wallet transactions
  • Other wallet actions

12.3 Security Safeguards

We implement multiple security measures to protect the Processing System, including:

Cryptographic Protections:

  • Hash verification of the WebView bundle before execution to ensure file integrity
  • Session-specific keypair generation with random nonces for mutual authentication
  • Encrypted message passing using ephemeral keys unique to each session
  • Replay and duplication protection through sequential identifiers and timestamps

Isolation Measures:

  • Restrictive Content Security Policy (CSP) to mitigate script injection and cross-site scripting
  • WebView execution in an isolated, incognito context
  • Debugging, file system access, caching, and third-party network requests disabled
